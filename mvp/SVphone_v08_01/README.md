# SVphone v06.01 - Media Transport Layer

**Phase 3 Implementation: Codec Negotiation, Quality Adaptation, and Encryption**

## Overview

SVphone v06.01 extends the call signaling from v06.00 with complete media transport capabilities including:

- **Codec Negotiation**: Opus audio + VP9/H.264 video with quality preferences
- **Quality Adaptation**: Dynamic bitrate and framerate adjustment based on network conditions
- **Media Encryption**: DTLS-SRTP for end-to-end encrypted media streams
- **Network Monitoring**: Real-time bandwidth and packet loss analysis

## Component Documentation

### 1. CodecNegotiation (`codec_negotiation.js`)

Handles codec selection and SDP negotiation for audio/video.

#### Supported Codecs

**Audio:**
- **Opus**: 6-510 kbps, 48kHz, 2 channels, 26.5ms latency
  - Features: DTX (discontinuous transmission), FEC (forward error correction)

**Video:**
- **VP9**: VP9 codec with temporal scalability
  - Bitrate: 100kbps-5Mbps
  - Framerate: 5-60 fps
  - Adaptive quality negotiation

- **H.264**: Baseline, main, high profiles
  - Bitrate: 100kbps-5Mbps
  - Framerate: 5-60 fps
  - Wide compatibility

#### Quality Presets

| Preset | Audio (kbps) | Video Bitrate | Resolution | Framerate |
|--------|--------------|---------------|-----------|-----------|
| **VHD** | 128 | 5 Mbps | 1920×1080 | 60 fps |
| **HD** | 64 | 2.5 Mbps | 1280×720 | 30 fps |
| **SD** | 48 | 500 kbps | 640×480 | 24 fps |
| **LD** | 24 | 100 kbps | 320×240 | 15 fps |

#### Key Methods

```javascript
const codecs = new CodecNegotiation({
  preferredAudioCodec: 'opus',
  preferredVideoCodec: 'vp9',
  preferredQuality: 'hd',
  maxBitrate: 5000000
})

// Create offer with codec preferences
const offer = await codecs.createOfferWithCodecs(peerConnection, {
  audioCodec: 'opus',
  videoCodec: 'vp9',
  quality: 'hd'
})

// Create answer with codec preferences
const answer = await codecs.createAnswerWithCodecs(peerConnection, {
  quality: 'hd'
})

// Analyze remote SDP
const analysis = codecs.analyzeRemoteSdp(remoteSdp)
// Returns: { audioCodec, videoCodec, audioBitrate, videoBitrate, ... }

// Get capabilities
const caps = codecs.getCodecCapabilities()

// Recommend quality based on bandwidth
const quality = codecs.recommendQuality(availableBitrate)
// Returns: 'vhd' | 'hd' | 'sd' | 'ld'
```

#### Events

- `codec:negotiated` - Codec selection completed
- `codec:mismatch` - Remote peer doesn't support preferred codec

### 2. QualityAdaptation (`quality_adaptation.js`)

Dynamically adjusts call quality based on network conditions.

#### Monitoring Metrics

- **Bandwidth**: Available outgoing bitrate (from GOOG REMB)
- **Packet Loss**: Percentage of lost packets
- **Jitter**: Variation in packet arrival times
- **Latency**: Round-trip time (RTT)
- **Video Bitrate**: Actual video transmission rate
- **Audio Bitrate**: Actual audio transmission rate

#### Adaptation Algorithm

```
Every 1 second:
  1. Collect RTC statistics
  2. Calculate moving averages (10-sample history)
  3. Check congestion indicators:
     - Packet loss > 5% → downgrade
     - Jitter > 100ms → downgrade
  4. Check bandwidth availability:
     - Available < 80% of quality needs → downgrade
     - Available > 150% of quality needs → upgrade
  5. Apply change (with 5s debounce to prevent oscillation)
```

#### Key Methods

```javascript
const quality = new QualityAdaptation({
  statsInterval: 1000, // Collect stats every 1s
  qualityChangeThreshold: 0.15, // 15% change triggers adjustment
  maxBitrate: 5000000,
  minBitrate: 100000,
  initialQuality: 'hd'
})

// Start monitoring
quality.startMonitoring(peerConnection, peerId)

// Listen for quality changes
quality.on('quality:changed', (data) => {
  console.log(`Quality changed from ${data.oldQuality} to ${data.newQuality}`)
  console.log(`Video bitrate: ${data.videoSettings.bitrate} bps`)
  console.log(`Framerate: ${data.videoSettings.framerate} fps`)
})

// Get current settings
const settings = quality.getCurrentSettings()
// Returns: { quality, video, audio, stats }

// Force quality to specific level
quality.forceQuality('hd')

// Stop monitoring
quality.stopMonitoring()
```

#### Events

- `quality:changed` - Quality level adjusted due to network conditions
- `quality:forced` - Quality manually set by application

### 3. MediaSecurity (`media_security.js`)

Manages DTLS-SRTP encryption for media streams.

#### Encryption Details

**DTLS (Datagram TLS):**
- Version: DTLS 1.2 (RFC 6347)
- Cipher Suites:
  - TLS_ECDHE_ECDSA_WITH_AES_128_GCM_SHA256 (preferred)
  - TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256
  - TLS_AES_128_GCM_SHA256 (TLS 1.3)

**SRTP (Secure RTP):**
- Profile: SRTP_AES128_CM_SHA1_80 (primary)
- Alternatives:
  - SRTP_AEAD_AES_128_GCM (modern, recommended)
  - SRTP_AEAD_AES_256_GCM (high security)

**Certificate:**
- Generated automatically by browser
- SHA-256 fingerprint verification
- Ephemeral keys (session-specific)

#### Key Methods

```javascript
const security = new MediaSecurity({
  encryptionEnabled: true,
  securityLevel: 'high' // low | medium | high
})

// Initialize security
await security.initialize()

// Verify remote fingerprint from SDP
const isValid = security.verifyFingerprint(peerId, remoteSdp)

// Monitor DTLS connection state
security.monitorDtlsState(peerConnection, peerId)

// Listen for security events
security.on('security:dtls-connected', (data) => {
  console.log('DTLS connection established:', data.peerId)
})

security.on('security:fingerprint-verified', (data) => {
  console.log(`Verified ${data.algorithm} fingerprint: ${data.hash}`)
})

// Check if call is encrypted
const encrypted = security.isEncrypted(peerConnection)

// Get security status
const status = security.getSecurityStatus()
// Returns: { encryptionEnabled, dtls, srtp, verifiedPeers }
```

#### Events

- `security:initialized` - Security layer initialized
- `security:fingerprint-verified` - DTLS fingerprint verified
- `security:dtls-connected` - DTLS handshake complete
- `security:dtls-failed` - DTLS connection failed

## Complete Call Flow with Media Transport

### Caller Side

```javascript
// 1. Initialize components
const codecs = new CodecNegotiation({ preferredQuality: 'hd' })
const quality = new QualityAdaptation()
const security = new MediaSecurity()

await security.initialize()

// 2. Initiate call (from v06.00)
const session = await callManager.initiateCall(calleeAddress, {
  audio: true,
  video: true,
  quality: 'hd',
  mintTokenFn: async (token) => { /* mint */ }
})

// 3. Create offer with codec preferences
const offer = await codecs.createOfferWithCodecs(peerConnection, {
  audioCodec: 'opus',
  videoCodec: 'vp9',
  quality: 'hd'
})

// Add security parameters to SDP
const secureSdp = security.addSecurityToSdp(offer.sdp)

// Send SDP to remote peer (via blockchain or signaling)

// 4. Receive answer
const remoteSdp = receivedAnswer.sdp

// Verify remote security
security.verifyFingerprint(calleeAddress, remoteSdp)
security.monitorDtlsState(peerConnection, calleeAddress)

// Analyze remote codec capabilities
const analysis = codecs.analyzeRemoteSdp(remoteSdp)

// 5. When connected, start quality monitoring
quality.on('quality:changed', (data) => {
  console.log(`Switched to ${data.newQuality}`)
  // Update UI to show current quality
})

quality.startMonitoring(peerConnection, calleeAddress)

// 6. Monitor call quality
callManager.on('call:stats-updated', (data) => {
  console.log('Video bitrate:', data.stats.video.outbound.bytesSent)
  console.log('Packet loss:', data.stats.video.inbound.packetsLost)
})

// 7. End call with final statistics
await callManager.endCall(session.callTokenId)
quality.stopMonitoring()
```

### Callee Side

```javascript

```

```
- Recommendation: Downgrade to SD

```

## Security Considerations

### DTLS-SRTP Advantages

1. **End-to-End Encryption**: Media encrypted peer-to-peer
2. **Mutual Authentication**: Certificate fingerprint verification
3. **Forward Secrecy**: Ephemeral keys per session
4. **Jitter Protection**: SRTP handles out-of-order packets

### Verification Steps

1. Extract fingerprint from remote SDP
2. Verify fingerprint SHA-256 hash format
3. Monitor DTLS handshake completion
4. Verify connection state == 'connected'

### Attack Prevention

- **Man-in-the-Middle**: DTLS handshake verifies identity
- **Replay Attacks**: SRTP sequence numbers prevent replay
- **Tampering**: HMAC authentication detects modifications
- **Eavesdropping**: AES-128/256 encryption prevents listening
